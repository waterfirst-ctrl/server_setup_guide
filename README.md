## 🛠️ 1장. 우분투 서버 기본 세팅 및 하드웨어 유지보수

노트북(Note PC)을 서버로 사용할 때 가장 중요한 것은 전원 관리와 프로세스 모니터링입니다.

* **[상황 1] 노트북 배터리 수명 및 전원 공급 상태를 모니터링하고 싶을 때**
* **[행동]** 우분투 터미널에 아래 명령어를 입력하여 배터리 충전 상태 및 하드웨어 인식을 체크합니다.
```bash
upower -i /org/freedesktop/UPower/devices/battery_BAT0

```




* **[상황 2] 서버에 가동 중인 전체 도커 컨테이너들의 구동 현황을 파악하고 싶을 때**
* **[행동]** 터미널에 아래 명령어를 입력하여 정상 작동 중인지(Up 상태) 구조를 파악합니다.
```bash
sudo docker ps -a
```

## 🌐 2장. VS Code 원격 터널(Remote Tunnels) 연결 및 제어

회사 보안 방화벽(22번 SSH 포트 차단)을 완벽하게 우회하여 집 서버를 내 모니터 앞으로 소환하는 핵심 핵심 통로입니다.

### 1. 집 우분투 서버에서 터널 생성 및 영구 서비스 등록

* **[상황 3] 서버 재부팅 등으로 터널 통로가 닫혔거나, 터널을 백그라운드에 영구 상주시키고 싶을 때**
* **[행동]** 우분투 본체 터미널에서 아래 명령어를 입력합니다.
```bash
# CLI 도구 실행 및 Microsoft/GitHub 계정 최초 연동
./code tunnel

# [인증 완료 후] 터널을 시스템 서비스로 등록하여 평생 백그라운드에서 자동 구동되게 설정
./code tunnel service install

```





### 2. 회사/외부 PC에서 집 서버 접속 방법

* **[상황 4] 회사 PC의 VS Code 앱에서 집 서버 폴더를 직접 제어하고 싶을 때**
* **[행동]** 회사 VS Code 로그인 및 연결 단계를 밟습니다.
1. VS Code 확장(`Ctrl + Shift + X`) 탭에서 **`Remote - Tunnels`** 설치
2. 왼쪽 하단 파란색 **`><` (원격 창 열기)** 클릭 ➡️ **`Connect to Tunnel...`** 선택
3. 집 서버 세팅 시 사용한 Microsoft 계정으로 로그인 후 **`server`** 클릭
4. 연결 후 [File] ➡️ [Open Folder]를 눌러 `/home/waterfirst/web` 경로 진입




* **[상황 5] 회사 PC에 프로그램 설치가 불가능하여 웹 브라우저로 긴급 제어해야 할 때**
* **[행동]** 크롬 브라우저 주소창에 아래 전용 URL을 입력하고 마이크로소프트 계정 로그인을 거쳐 웹 브라우저형 에디터로 진입합니다.
```text
https://vscode.dev/tunnel/server

```

## 🚀 3장. 포트(Port)별 다중 서비스 배포 및 인프라 마스터 리셋

도커(Docker) 컨테이너 구조를 활용해 포트 번호를 나누어 독립된 서비스를 무한대로 확장하는 실전 레이아웃입니다.

### 1. 기본 80번 포트 웹 배포 및 마스터 치유 루틴

* **[상황 6] 메인 웹페이지(`index.html`) 코드를 전면 수정하여 배포하고 싶을 때**
* **[행동]** 원격 터미널에서 아래 `cat` 스크립트를 열고, 내부에 수정된 HTML 코드를 마우스 우클릭으로 붙여넣어 파일을 갱신합니다.
```bash
sudo cat << 'EOF' > /home/waterfirst/web/index.html
<!-- 여기에 최신 HTML/CSS 대시보드 코드를 붙여넣으세요 -->
EOF

```




* **[상황 7] 코드를 반영했는데 화면 갱신이 안 되거나 도커 내부 인프라를 리셋하고 싶을 때**
* **[행동]** 컨테이너 찌꺼기를 안전하게 완전히 밀어버리고 재생성하는 미리 짜둔 마스터 치유 스크립트를 원터치 실행합니다.
```bash
/home/waterfirst/deploy.sh

```





### 2. `8500`번 등 멀티 포트 서브 서비스 확장 루틴

* **[상황 8] 기본 웹과 완전히 분리된 별도의 분석용 툴(예: R Shiny, DB 뷰어)을 특정 포트에 배포하고 싶을 때**
* **[행동]** 새로운 포트 폴더를 만들고 도커 포트 바인딩(`-p`) 설정을 분리하여 컨테이너를 하나 더 생성합니다. (`image_690925.png` 실증 완료)
```bash
# 1. 포트 전용 독립 작업 디렉토리 생성
mkdir -p /home/waterfirst/web_8500

# 2. 해당 폴더에 고유 서브 콘텐츠 배치
sudo cat << 'EOF' > /home/waterfirst/web_8500/index.html
<!-- 8500번 포트용 독립 웹 파일 코드 입력 -->
EOF

# 3. 8500번 포트를 움켜쥐는 고유 Nginx 컨테이너 독립 기동
sudo docker run -d \
  --name web-server-8500 \
  -p 8500:80 \
  -v /home/waterfirst/web_8500:/usr/share/nginx/html \
  --restart always \
  nginx:alpine

```




* **[확인]** 외부 브라우저에서 `[http://221.153.204.191:8500](http://221.153.204.191:8500)`으로 접속하여 독립된 화면 구조를 검토합니다.
